<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Visitor → site pública | Nenhuma entrada de usuário; apenas leitura de conteúdo estático |
| Client → WhatsApp externo | Link `wa.me` redireciona para fora; sem dados sensíveis no query |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | I (Information Disclosure) | WhatsAppFloatingButton | mitigate | `target="_blank"` + `rel="noopener noreferrer"` em todos links externos — previne window.opener tampering |
| T-01-02 | T (Tampering) | lib/site-config.ts | accept | WHATSAPP_NUMBER e SITE_NAME são constantes server-side, não vêm de user input |
| T-01-03 | D (Denial of Service) | Header + MobileNav | accept | Zero-dependência runtime, RSC compila estático; sem vetor de DoS |
</threat_model>
